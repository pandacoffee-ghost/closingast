# 轻量私人衣橱系统设计文档

- 日期：2026-03-20
- 状态：Approved
- 目标：为个人或家庭用户提供一个轻量、手机优先、可云端轻部署的衣橱管理系统，核心解决“记住自己有哪些衣服”和“减少重复购买”两个问题。

## 1. 产品概述

这是一个面向手机端的私人衣橱系统。用户可以通过拍照上传已有衣服图片，或从淘宝、京东等平台粘贴商品链接半自动导入衣物信息。系统以图片卡片为核心展示形式，帮助用户快速浏览、筛选、回忆自己已有的衣服，并对相似单品进行轻提示。

系统强调“小而可用”：

- 不做原生 App，优先移动端 Web
- 不做复杂电商登录同步
- 不做订单历史全量抓取
- 不做重型 AI 穿搭系统
- 适合个人或小规模家庭用户部署

## 2. 目标用户与核心问题

### 目标用户

- 经常在淘宝、京东等平台购买衣服的个人用户
- 主要通过手机管理生活物品
- 更需要视觉浏览和轻量整理，而非精细财务统计

### 核心问题

- 衣服买得多，记不住自己已有的款式
- 新买衣服前，不确定家里是否已有相似款
- 不同平台购买记录分散，难以统一查看
- 传统清单工具以文字为主，不适合衣物管理

## 3. 产品目标与非目标

### 产品目标

- 用户可在 3 秒内通过图片流和筛选快速找到某件衣服
- 新增一件衣服的操作尽量控制在 30 秒内
- 即使商品链接解析失败，也能顺畅完成手动保存
- 第一版即可形成持续录入习惯，而不是一次性整理工具

### 非目标

- 不支持淘宝、京东账号登录态同步
- 不支持历史订单批量自动拉取
- 不做复杂搭配推荐或 AI 造型建议
- 不做多用户协作和后台管理系统
- 不做库存、尺码、退换货等 ERP 化能力

## 4. 方案选择

本项目采用“衣橱卡片库”方案。

## 5. MVP 功能范围

### MVP 包含

- 手机端登录
- 衣物图片上传
- 淘宝/京东商品链接导入
- 衣橱图片流浏览
- 按类目、季节、颜色、场景筛选
- 衣物详情查看与编辑
- 简单相似单品提醒
- 闲置标记

### 后续阶段再考虑

- 商品截图 OCR
- AI 自动打标签
- 搭配建议
- 闲置统计和清理提醒
- 小程序形态

## 6. 信息架构

一级导航控制在 4 个入口：

- 衣橱
- 新增
- 发现
- 我的

## 7. 核心用户流程

### 流程一：查看已有衣服

1. 用户打开首页
2. 浏览图片流或使用筛选
3. 点进详情查看来源、购买信息、备注和相似款

### 流程二：新增衣服

1. 用户点击新增
2. 选择拍照上传或粘贴商品链接
3. 系统尝试自动补全标题、主图、平台和价格
4. 用户补充必要标签
5. 保存成功后进入衣橱

### 流程三：避免重复购买

1. 用户上传图片或商品链接
2. 系统根据类目、颜色、标题关键词做相似匹配
3. 在确认页或详情页给出轻提示
4. 用户自行判断是否重复

## 9. 导入策略

- 图片上传是稳定主入口
- 商品链接导入只做半自动解析，失败时允许手动补录
- 商品截图导入第一版只保存图片，不强依赖 OCR

## 10. 技术架构

- 前端：Next.js
- 服务端接口：Next.js API Routes 或 Server Actions
- 数据库：Supabase Postgres
- 对象存储：Supabase Storage
- 鉴权：Supabase Auth
- 部署：Vercel

## 11. 成功标准

- 可稳定录入 100 到 500 件衣物
- 手机端浏览和筛选顺畅
- 用户愿意持续录入
- 能明显降低“是否买过类似款”的不确定感
